# 가족 어드벤트 캘린더 — 해커톤 D-Day 실행 플랜

## Context

Indie Crew 1st Hackathon (1인 / 2시간 / "가정의 달" 주제) 출품작.

**최종 컨셉 (확정)**: 가족이 함께 채우는 5월 한 달 어드벤트 캘린더. 매일 한 가지 주제, 추천 질문 3개. 가족 구성원이 각자 1개를 골라 답함. **양방향 + 다중 멤버 공동 작성**.

**컨셉 진화 흐름**
1. 1차안: 자녀 → 부모님 일방향 (자녀 큐레이션, 부모님 매일 한 칸 봄)
2. 2차안 (확정): **가족 구성원 모두 채움**. 매일 같은 주제·다른 답변. (= 발산 단계 후보 #11 가족 Q&A 30일 + #15 동시답변 일치 + 어드벤트 페이싱의 결합)
3. 3차 refine (이번 turn): 워크플로우 디테일 — 진입 흐름·날짜 잠금·데모 화살표·모바일 우선 명세

**왜 이 방향이 강한가**
- 진심의 출처 = *가족 각자가 본인 손으로 적은 답*. AI 생성 X.
- "부모님이 더 잘 아시는 영역"을 *오히려 컨셉으로 흡수* — 같은 질문에 다른 시점의 답이 나란히.
- 30초 데모 임팩트: 5/8 어버이날 칸 = 엄마·아빠·딸 답변 나란히.
- 사용자 본인이 "내년에도 무조건 운영" 발언 → V1을 V2 시드로 설계.

## V1 (해커톤 2시간) vs V2 (내년 실서비스) 스코프

| 영역 | V1 | V2 |
|---|---|---|
| 작성 방향 | **양방향 공동 작성** ✓ | 답변에 댓글·이모지 리액션 |
| 가족 구성원 | 캘린더 생성 시 명단 입력. **개별 이름** (엄마/아빠/딸 등) | 초대 링크 별도, 권한 관리 |
| 멤버 식별 | URL 진입 시 멤버 선택 + localStorage | 진짜 인증 |
| 콘텐츠 | 주제 + 추천 질문 3개 중 선택, 텍스트 + 사진 1장 | + 음성/영상, 자유 질문 |

| 마무리 | 단일 페이지 인쇄 친화 뷰 (`window.print()`) | PDF/책 출력, 인쇄 발송 |
| 답변 수정 | **불가 (V1 단순화)** | 가능 |
| 플랫폼 | **모바일 우선** | + 데스크톱 최적화 |

## V1 핵심 기능

1. **캘린더 생성** — 캘린더 제목 + 가족 구성원 이름 N개 입력 → URL slug 발급
2. **멤버 진입** — 초대 링크 → "당신은 누구신가요?" → 멤버 카드 선택 → localStorage 저장 (다음부터 자동)
3. **31일 월간 캘린더 그리드** — 6주 × 7일 형식. 각 칸 시각화(이모지/카네이션/색 점/사진 썸네일)
4. **칸 열기** — 5/1~devNow까지만 클릭 가능. 그 날의 주제 + 추천 질문 3개 + (자유 입력) → 답변 작성
5. **멀티 답변 표시** — 같은 칸에 가족 모두의 답변이 멤버 색·이름 라벨로 나란히
6. **데모용 devNow 시뮬레이터** — 숨김 화살표 버튼으로 가상 오늘을 어제/내일로 1일씩 이동
7. **요약(Book) 뷰** — 31칸 한 페이지 그리드. `Cmd+P`로 PDF 저장
8. **시드 데이터 1세트** — 데모용 캘린더 1개 (멤버 3명, 8일치 답변)

- 데스크톱 전용 UI (모바일 뷰 가운데 정렬로 통일)

## 워크플로우 (V1 데모 명세) ← 사용자 요청 4개 항목

### 1️⃣ 진입 흐름 — 초대 링크 → 멤버 선택

- 초대 링크: `https://<도메인>/?id=<slug>`
- **첫 진입**: "당신은 누구신가요?" 화면
  - 캘린더 멤버 카드로 표시 (각 카드에 이름 + 자동 할당 색)
  - 클릭 → `localStorage.setItem('member_id_' + slug, memberId)` → 캘린더로 이동
- **두 번째 진입**: localStorage 읽어 자동으로 캘린더 화면 (멤버 선택 화면 스킵)
- **로그아웃 / 멤버 변경**: V1 미지원. localStorage 수동 삭제로만 가능 (데모 시 시크릿 모드 활용)

### 2️⃣ 날짜 접근 룰

- **5/1 ~ devNow** 까지만 클릭 가능 (모달 열림 + 답변 작성)
- **미래 날짜**: 흐림 + 자물쇠 아이콘. 클릭 시 미니 토스트 "아직 열리지 않았어요"
- **지난 날짜**: 클릭 가능. 답변 *작성*은 가능 (안 채운 사람), *수정·삭제는 V1에서 불가*
- **5월 밖**: 4월·6월 칸은 표시되지만 dim. 클릭 비활성

### 3️⃣ 데모용 devNow 시뮬레이터 (숨김 화살표)

- **목적**: 발표 시 "오늘은 5/8" / "오늘은 5/21" 시뮬레이션 가능하게
- **노출**: `?dev=1` URL 파라미터일 때만 표시 (일반 사용자 안 보임)
- **위치**: 캘린더 헤더 우측, 작은 ◀ ▶ 버튼 두 개 (또는 햄버거 메뉴 안)
- **동작**:
  - ◀ → devNow 하루 빼기. ▶ → 하루 더하기
  - 5/1 ~ 5/31 범위로 클램프
  - 변경 시 캘린더 잠금/언락 즉시 반영
- **저장**: `localStorage.devNow_<slug>` (실수 새로고침 안전)
- **시각**: `?dev=1`일 때 헤더에 작은 "DEV" 배지 표시

### 4️⃣ 모바일 우선 UI

- **타깃**: iPhone SE(375px) ~ Galaxy 표준. 데스크톱은 모바일 뷰 가운데 정렬 (`max-width: 480px`)
- **월간 캘린더 형식**: 6주 × 7일 그리드
  - 5/1 = 금요일 → 첫 주 일·월·화·수·목 4칸 빈 자리 (또는 직전 4월 말 dim)
  - 5/31 = 일요일
  - 총 42칸 (5월 외 11칸 dim/숨김)
- **각 칸의 시각 표현** (사용자 요청 — *예쁘게 꾸미기*):
  | 상태 | 시각 |
  |---|---|
  | 빈 칸 (5월 미래) | 흐린 회색 + 자물쇠 |
  | 빈 칸 (5월 오늘 이전, 아무도 답 안 함) | 흐린 회색 + 점선 동그라미 |
  | ★ 기념일 칸 | 그 날 전용 이모지/아이콘 (예: 5/5 🧒, 5/8 🌹 카네이션, 5/15 👨‍👩‍👧 가족, 5/19 🎓, 5/21 💑) |
  | 답변 1명 | 멤버 색 점 1개 |
  | 답변 N명 | 멤버 색 점 N개 (작은 도트 그룹) |
  | 사진 첨부 답변 있음 | 칸 배경에 썸네일 살짝 (블러 + 색 오버레이) |
  | 오늘 (devNow) | 굵은 테두리 + 살짝 큼 + 발광 |
- **칸 클릭 → 모달**: 화면 전체화면 모달 (모바일이라 fullscreen sheet 형식)

→ 칸 자체가 *정보 시각화*. 모바일 작은 화면에서도 한눈에 누가 채웠나 확인 가능.

## 기술 아키텍처

| 레이어 | 선택 | 이유 |
|---|---|---|
| Frontend | Vite + Svelte 5 + TS + SCSS | 이미 세팅 (frontend/) |
| 라우팅 | `?id=xxx&mode=create\|book` + `?dev=1` query string | 라우터 라이브러리 X |

| PDF | `window.print()` + `@media print` | 라이브러리 X |

**라이브러리 추가 가능성**: V1 기준 *없음*. 짧은 ID는 자작 (`Math.random().toString(36).slice(2,10)`).

## DB 스키마 (테이블 2개 — STRATEGY 준수)

```sql
-- calendars
create table calendars (
  id uuid primary key default gen_random_uuid(),
  slug text unique not null,                -- 8자 식별자
  title text not null,                      -- "우리집 5월"
  members jsonb not null default '[]',      -- [{id, name, color}, ...]
  year int not null default 2026,
  created_at timestamptz default now()
);

-- entries
create table entries (
  id uuid primary key default gen_random_uuid(),
  calendar_id uuid references calendars(id) on delete cascade,
  member_id text not null,                  -- members[i].id
  date date not null,
  question text not null,                   -- 선택한 추천 질문 (또는 자유 입력)
  body text,
  image_url text,
  created_at timestamptz default now(),
  unique(calendar_id, member_id, date)
);

-- Storage bucket: "entries" (public read)
```

**주제·추천 질문은 DB에 안 넣음** — `frontend/src/lib/topics.ts` 클라이언트 상수.

**members.color 자동 할당**: lavender / peach / sage / sky / butter — 5가지 팔레트 순환.

**미션 칸 이모지 매핑** (`topics.ts`에 함께):
```ts
'2026-05-01': { ..., emoji: '🛠️' },  // 근로자의 날
'2026-05-05': { ..., emoji: '🧒' },  // 어린이날
'2026-05-08': { ..., emoji: '🌹' },  // 어버이날 카네이션
'2026-05-15': { ..., emoji: '👨‍👩‍👧' }, // 가정의 날
'2026-05-19': { ..., emoji: '🎓' },  // 성년의 날
'2026-05-21': { ..., emoji: '💑' },  // 부부의 날
'2026-05-31': { ..., emoji: '✉️' },  // 클라이맥스
```

## 화면 흐름

```
[/]                         랜딩 — 카피 + [캘린더 만들기]
  │
  └─→ [/?mode=create]       제목 + 멤버 이름들 → calendars insert → slug 발급
       │
       └─→ [/?id=xxx]       (멤버 미선택 시) "당신은 누구신가요?"
            └─ 멤버 카드 클릭 → localStorage 저장
            │
            └─→ [/?id=xxx]  월간 캘린더 (6주 × 7일)
                 ├─ devNow 칸 강조, 5/1~devNow 클릭 가능
                 ├─ 미션 칸 이모지, 답변자 색 점, 사진 썸네일
                 ├─ ?dev=1 면 헤더에 ◀ ▶ 버튼 (devNow 이동)
                 │
                 └─→ 칸 탭 → 풀스크린 시트
                      ├─ 주제 한 줄 + 추천 질문 3개
                      ├─ 1개 선택 (or 자유 입력) + 본문 + 사진 → 저장
                      └─ 다른 멤버 답변 함께 표시 (색·이름 라벨)
            │
            └─→ [/?id=xxx&mode=book]  요약 뷰 → Cmd+P
```

## 31일 주제와 추천 질문

각 날 = **주제 한 줄 + 추천 질문 3개**. 가족 구성원은 1개 골라서 답하거나 자유 입력.

> **양립성 원칙**: 모든 추천 질문은 *부모님이든 자녀든 누가 답해도* 자연스러운 형태.

### 5월 1주차

| 날 | 요일 | 분류 | 주제 | 추천 질문 3개 |
|---|---|---|---|---|
| 5/1 | 금 | ★ 근로자의 날 🛠️ | **일하는 마음** | • 최근 가장 힘들었던 일은? • 오늘 출퇴근 길의 풍경은? • 어릴 적 일하는 어른의 모습 한 장면? |
| 5/2 | 토 | ☆ | **5월의 첫 주말** | • 오늘 가장 좋았던 순간은? • 주말 아침 가장 먼저 한 일은? • 오늘 점심 메뉴는? |
| 5/3 | 일 | ☆ | **쉼** | • 최근 가장 푹 쉰 날은? • 쉴 때 가장 행복한 모습은? • 오늘 듣고 싶은 노래 한 곡은? |
